  <behavior>
    - addVocabToPool(['voc-001', 'voc-002']): creates 2 SrsCard entries with createEmptyCard() state
    - addVocabToPool(['voc-001']): does NOT overwrite existing card (idempotent)
    - recordReview('voc-001', Rating.Good): updates card via fsrs().repeat(), card.state transitions from New to Learning
    - recordReview for a New card increments newCardsToday and sets newCardsDayStamp to today
    - recordReview for a Review card does NOT increment newCardsToday
    - newCardsToday resets when newCardsDayStamp !== today's date
    - reviewedTodayCount increments on every review and resets on new day
    - After JSON serialize/deserialize round-trip: card.due is a Date object (not string)
    - Corruption handler: invalid JSON in localStorage sets _corruptionDetected = true and resets cards to {}
  </behavior>
  <action>
Follow the RESEARCH.md Pattern 1 (SRS Store Schema) and Pattern 2 (FSRS Scheduling) exactly. Follow the existing useProgressStore pattern for Zustand persist setup.

Create `src/store/srs.ts`:
- Interface `SrsCard { vocabId: string; card: Card }` where Card is from ts-fsrs
- Interface `SrsStoreState` with: cards (Record keyed by vocabId), newCardsToday, newCardsDayStamp, reviewedTodayCount, reviewedDayStamp, _corruptionDetected, and actions: addVocabToPool, recordReview, clearCorruptionFlag
- `addVocabToPool(vocabIds: string[])`: only add cards that don't already exist (use createEmptyCard())
- `recordReview(vocabId: string, rating: Rating)`: call `fsrs().repeat(entry.card, new Date())`, store `scheduling[rating].card`. Track daily new card count and review count with day stamp comparison.
- persist config: name 'hanbuddy_srs', version 1, partialize excludes actions and _corruptionDetected
- migrate: version < 1 returns defaults. Version 1 rehydrates Date objects on card.due and card.last_review (JSON serializes Date as string — must convert back). See RESEARCH.md Pitfall 4.
- onRehydrateStorage: on error, remove localStorage key, set _corruptionDetected and reset cards

Create `src/store/srs.test.ts` following the TDD behavior list above. Use vi.useFakeTimers() for date-dependent tests. Reset localStorage between tests.

IMPORTANT: Use day stamp format `new Date().toISOString().slice(0, 10)` for day boundary comparisons per RESEARCH.md anti-pattern guidance.
